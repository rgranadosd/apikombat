                    REGLAS COMPLETAS DEL JUEGO - API KOMBAT

OBJETIVO DEL JUEGO
================================================================================
El objetivo es ser el primer jugador en tener 4 aspectos saludables (no 
vulnerables) en tu tablero. Los aspectos saludables son aquellos que no tienen 
el estado "vulnerable" activado.

COMPONENTES DEL JUEGO
================================================================================

JUGADORES
---------
- 2 jugadores (TÚ vs IA)
- Cada jugador tiene:
  * Una mano de cartas (máximo ilimitado, pero se mantienen 3 cartas al final 
    de cada turno)
  * Un tablero con slots para aspectos (máximo 4 aspectos simultáneos)
  * Un escudo de Code Freeze (opcional, se activa con una carta especial)

MAZO DE CARTAS
--------------
Total: 89 cartas

1. ASPECTOS (SCORING) - 17 cartas
   - 4 copias de cada aspecto:
     * Seguridad (SEGURIDAD)
     * Documentación (DOCUMENTACIÓN)
     * Gobierno (GOBIERNO)
     * Performance (PERFORMANCE)
   - 1 carta Wildcard (multicolor)

2. ATAQUES Y PROBLEMAS - 29 cartas
   - 16 cartas de ATAQUE de Seguridad (4 copias de cada):
     * DoS
     * SQL Injection
     * OWASP
     * Brute-Force
   - 12 cartas de PROBLEMA (4 copias de cada):
     * Problema DOCUMENTACIÓN
     * Problema GOBIERNO
     * Problema PERFORMANCE
   - 1 carta Wildcard (Ataque/Problema multicolor)

3. PROTECCIONES - 29 cartas
   - 16 cartas de PROTECCIÓN de Seguridad (4 copias de cada):
     * OAuth2/JWT
     * HTTPS/SSL
     * Validación de Entrada
     * Rate Limiting
   - 12 cartas de PROTECCIÓN genérica (4 copias de cada):
     * Protección DOCUMENTACIÓN
     * Protección GOBIERNO
     * Protección PERFORMANCE
   - 1 carta Wildcard (Protección multicolor)

4. INTERVENCIONES - 14 cartas
   - 4 copias de Refactoring
   - 4 copias de Migración
   - 4 copias de Activo-Activo
   - 1 copia de Code Freeze
   - 1 copia de Rollback

ESTADOS DE LOS ASPECTOS
================================================================================
Cada aspecto en el tablero puede tener dos propiedades:

1. VULNERABLE (vulnerable: true/false)
   - Un aspecto vulnerable puede ser destruido por un ataque
   - Si un aspecto vulnerable recibe un ataque, se elimina del tablero
   - Estado visible: aspecto con marca visual de vulnerabilidad

2. PROTECCIONES (protecciones: 0-2)
   - Nivel 0: Sin protección
   - Nivel 1: Protegido (bloquea ataques)
   - Nivel 2: Fortalecido (bloquea ataques y no puede ser afectado por 
     Activo-Activo)

TIPOS DE CARTAS Y SUS EFECTOS
================================================================================

1. ASPECTOS (tipo: "aspecto" o "scoring")
-----------------------------------------
Acción: Colocar un nuevo aspecto en tu tablero

Reglas:
- Solo puedes tener un aspecto de cada color (seguridad, documentación, 
  gobierno, performance)
- No puedes tener más de 4 aspectos en total
- Si juegas un aspecto multicolor, se resuelve automáticamente al primer color 
  disponible que no tengas
- Los aspectos se colocan saludables (no vulnerables, 0 protecciones)

Restricciones:
- No puedes jugar un aspecto si ya tienes ese color
- No puedes jugar un aspecto si ya tienes 4 aspectos

2. ATAQUES Y PROBLEMAS (tipo: "ataque", "problema" o "virus")
---------------------------------------------------------------
Acción: Atacar un aspecto del rival

Reglas:
- El objetivo debe ser el oponente
- El aspecto objetivo debe existir en el tablero del rival
- Si el rival tiene Code Freeze activo, el ataque se bloquea y consume el 
  escudo (el ataque no tiene efecto)
- Si el aspecto objetivo tiene 1 o más protecciones, el ataque se bloquea 
  (sin efecto)
- Si el aspecto objetivo NO tiene protecciones:
  * Si está saludable: se marca como vulnerable
  * Si ya está vulnerable: se destruye (elimina del tablero)

Restricciones:
- No puedes atacar si el rival no tiene el aspecto objetivo
- No puedes atacar si el aspecto objetivo está protegido
- No puedes atacar si el aspecto objetivo está fortalecido (2 protecciones)

3. PROTECCIONES (tipo: "proteccion" o "medicina")
--------------------------------------------------
Acción: Proteger o fortalecer un aspecto tuyo

Reglas:
- El objetivo debe ser un aspecto en tu propio tablero
- Si el aspecto está vulnerable, primero lo cura (elimina vulnerabilidad)
- Luego aumenta el nivel de protección:
  * Si tenía 0 protecciones → pasa a 1 (Protegido)
  * Si tenía 1 protección → pasa a 2 (Fortalecido)
  * Si ya tenía 2 protecciones → se mantiene en 2 (máximo)

Restricciones:
- No puedes proteger si no tienes el aspecto objetivo
- No puedes proteger más allá de 2 protecciones (máximo)

4. INTERVENCIONES (tipo: "intervencion" o "tratamiento")
----------------------------------------------------------
Las intervenciones tienen efectos especiales según su subtipo:

A) MIGRACIÓN (subtipo: "migracion" o "ladrón")
-----------------------------------------------
Acción: Robar un aspecto del rival

Reglas:
- Roba el primer aspecto del rival que tú no tengas
- Si se especifica un color objetivo, roba ese aspecto específico
- El aspecto robado mantiene su estado (vulnerable/protecciones)
- Si el rival tiene Code Freeze activo, la migración se bloquea y consume 
  el escudo

Restricciones:
- No puedes robar si el rival no tiene aspectos que tú no tengas
- No puedes robar un aspecto que ya tienes
- No puedes robar si el rival no tiene el aspecto objetivo específico

B) REFACTORING (subtipo: "refactoring" o "trasplante")
------------------------------------------------------
Acción: Intercambiar un aspecto tuyo con uno del rival

Reglas:
- Intercambia un aspecto tuyo con uno del rival
- Si se especifica un color objetivo del rival, intercambia ese aspecto
- Si no se especifica, intercambia el primer aspecto de cada jugador
- Los aspectos mantienen su estado (vulnerable/protecciones)
- Si el rival tiene Code Freeze activo, el refactoring se bloquea y consume 
  el escudo

Restricciones:
- Ambos jugadores deben tener al menos un aspecto
- No puedes intercambiar si el rival no tiene el aspecto objetivo específico
- Si ambos tienen el mismo aspecto, no hay efecto

C) ACTIVO-ACTIVO (subtipo: "activo_activo" o "contagio")
---------------------------------------------------------
Acción: Vulnerar un aspecto del rival que tenga el mismo color que uno tuyo 
        vulnerable

Reglas:
- Debes tener al menos un aspecto vulnerable
- El rival debe tener el mismo aspecto (mismo color) que uno de tus aspectos 
  vulnerables
- El aspecto del rival no debe estar fortalecido (2 protecciones)
- El aspecto del rival no debe estar ya vulnerable
- Si cumple las condiciones, vulnera el aspecto del rival
- Si el rival tiene Code Freeze activo, se bloquea y consume el escudo

Restricciones:
- Debes tener al menos un aspecto vulnerable
- El rival debe tener el mismo aspecto que uno de tus aspectos vulnerables
- El aspecto del rival no debe estar fortalecido ni ya vulnerable

D) CODE FREEZE (subtipo: "code_freeze" o "guante")
---------------------------------------------------
Acción: Activar un escudo que bloquea el próximo ataque o intervención

Reglas:
- Activa un escudo en tu jugador
- El escudo bloquea el próximo ataque o intervención dirigida contra ti
- Al bloquear, el escudo se consume (se desactiva)
- El escudo solo se consume si el ataque/intervención te afectaría

Restricciones:
- Ninguna (siempre jugable)

E) ROLLBACK (subtipo: "rollback" o "error")
--------------------------------------------
Acción: Reducir una protección del rival

Reglas:
- Reduce en 1 el nivel de protección de un aspecto del rival
- Si el aspecto tenía 1 protección, pasa a 0
- Si el aspecto tenía 2 protecciones, pasa a 1
- Si el aspecto no tenía protecciones, no tiene efecto
- Si el rival tiene Code Freeze activo, se bloquea y consume el escudo

Restricciones:
- El rival debe tener al menos un aspecto con protecciones (>= 1)

FLUJO DEL JUEGO
================================================================================

1. PREPARACIÓN
--------------
- Se crea el mazo completo (89 cartas)
- Se baraja el mazo
- Se reparten 3 cartas a cada jugador
- Se determina aleatoriamente quién empieza

2. TURNO DE UN JUGADOR
----------------------
Durante su turno, un jugador puede:

a) JUGAR UNA CARTA
   - Selecciona una carta de su mano
   - Verifica si es jugable (cumple restricciones)
   - Si es jugable, la juega y ejecuta su efecto
   - La carta jugada va al descarte
   - Después de jugar, el turno pasa al siguiente jugador

b) DESCARTAR CARTAS
   - Puede descartar 1 o más cartas de su mano
   - Las cartas descartadas van al descarte
   - Después de descartar, el turno pasa al siguiente jugador

c) PASAR TURNO
   - Si no puede o no quiere jugar/descartar, pasa el turno

3. FIN DEL TURNO
----------------
Al finalizar el turno:
- El jugador roba cartas hasta tener 3 cartas en su mano
- Si el mazo está vacío, se recicla el descarte (se baraja y se convierte 
  en el nuevo mazo)
- Si no hay cartas disponibles, el jugador continúa con las que tiene
- El turno pasa al siguiente jugador

4. VERIFICACIÓN DE VICTORIA
----------------------------
Después de cada acción, se verifica:
- Si un jugador tiene 4 aspectos saludables (no vulnerables), gana la partida
- El juego termina inmediatamente

CARTAS MULTICOLOR (WILDCARDS)
================================================================================
Las cartas multicolor pueden usarse como cualquier color, pero deben resolverse 
automáticamente:

1. ASPECTOS MULTICOLOR
   - Se resuelve al primer color disponible que el jugador no tenga
   - Si ya tiene los 4 aspectos, no se puede jugar

2. ATAQUES/PROBLEMAS MULTICOLOR
   - Se resuelve al primer aspecto que el rival tenga y que no esté protegido
   - Si todos los aspectos del rival están protegidos, no se puede jugar

3. PROTECCIONES MULTICOLOR
   - Se resuelve al primer aspecto que el jugador tenga
   - Si no tiene aspectos, no se puede jugar

RECICLAJE DEL DESCARTE
================================================================================
Cuando el mazo se agota:
- Todas las cartas del descarte se barajan
- El descarte se convierte en el nuevo mazo
- El descarte queda vacío
- Se continúa robando del nuevo mazo

ESTADOS ESPECIALES
================================================================================

CODE FREEZE (Escudo)
--------------------
- Se activa jugando la carta "Code Freeze"
- Bloquea el próximo ataque o intervención dirigida contra el jugador
- Se consume automáticamente al bloquear
- Solo protege contra ataques/intervenciones que afectarían al jugador
- No protege contra acciones que no te afectan directamente

PROTECCIONES DE ASPECTOS
-------------------------
- Nivel 1 (Protegido): Bloquea cualquier ataque/problema
- Nivel 2 (Fortalecido): Bloquea ataques y no puede ser afectado por 
  Activo-Activo
- Las protecciones no se acumulan más allá de 2

VULNERABILIDAD
--------------
- Un aspecto vulnerable puede ser destruido por un ataque
- Si un aspecto vulnerable recibe un ataque, se elimina del tablero
- Un aspecto vulnerable puede ser curado con una protección

CONDICIONES DE VICTORIA
================================================================================

VICTORIA
--------
Un jugador gana cuando:
- Tiene exactamente 4 aspectos saludables (no vulnerables) en su tablero
- La verificación se hace después de cada acción
- El juego termina inmediatamente al alcanzar la condición

EMPATE / BLOQUEO
----------------
- Si ambos jugadores tienen 4 aspectos saludables simultáneamente, gana el 
  jugador que alcanzó la condición primero (el que jugó la carta que lo llevó 
  a 4)
- Si después de 50 turnos consecutivos ningún jugador puede jugar cartas 
  válidas, la partida se considera bloqueada

RESTRICCIONES GENERALES
================================================================================

1. LÍMITE DE ASPECTOS
   - Máximo 4 aspectos simultáneos en el tablero
   - No puedes colocar un aspecto si ya tienes 4
   - No puedes tener dos aspectos del mismo color

2. MANO DE CARTAS
   - No hay límite máximo de cartas en la mano
   - Al final de cada turno, se roban cartas hasta tener 3
   - Si no hay cartas disponibles, se mantiene la mano actual

3. VALIDACIÓN DE JUGADAS
   - Todas las cartas deben cumplir sus restricciones para ser jugables
   - Si una carta no es jugable, no se puede jugar
   - El jugador debe descartar o pasar el turno si no puede jugar

4. ORDEN DE EFECTOS
   - Los efectos se resuelven inmediatamente al jugar la carta
   - Si un efecto falla, la carta ya se ha jugado (va al descarte)
   - Los efectos de protección se aplican antes de los ataques

NOTAS TÉCNICAS
================================================================================

- El juego usa un sistema de turnos alternos
- Los jugadores siempre tienen la oportunidad de jugar o descartar
- El reciclaje del descarte es automático cuando el mazo se agota
- La verificación de victoria es inmediata después de cada acción
- Los efectos de las cartas son atómicos (se ejecutan completamente o no)
